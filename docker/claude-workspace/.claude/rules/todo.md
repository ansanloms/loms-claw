# TODO 管理

ワークスペースルートの `TODO.md` でタスクを管理する。
メモリ（MEMORY.md / memory/）とは別物。記憶ではなく行動の管理。

## 保存先

`TODO.md`（ワークスペースルート）

## 形式

Markdown テーブルで管理する。

```markdown
# TODO

| 概要         | 期限       | ステータス | 備考            |
| ------------ | ---------- | ---------- | --------------- |
| やること概要 | 2026-04-05 | ⬜         | 補足があれば    |
| 別のやること | なし       | 🔵         |                 |
| 終わったやつ | なし       | ✅         | 2026-03-25 完了 |
```

### カラム定義

| カラム     | 必須 | 説明                                   |
| ---------- | ---- | -------------------------------------- |
| 概要       | yes  | 1 行で収まる簡潔な説明                 |
| 期限       | yes  | `YYYY-MM-DD` または `なし`             |
| ステータス | yes  | 下表の絵文字で表現                     |
| 備考       | no   | 補足・経緯・関連情報。空でも可         |

### ステータス

| 絵文字 | 意味   |
| ------ | ------ |
| ⬜     | 未着手 |
| 🔵     | 進行中 |
| ✅     | 完了   |
| ❌     | 中止   |

## 運用ルール

- ユーザから TODO の追加・更新・削除の指示があったら即座に反映しろ。
- ✅・❌ のタスクは完了・中止から 1 ヶ月経過後に削除して構わない。備考欄の日付で判断しろ。
- ユーザに期限を聞かれていない限り、勝手に期限を設定するな。`なし` にしろ。
- ステータス変更時は備考に日付と理由を追記しろ（例: `2026-04-01 完了`）。
- TODO の参照・更新が必要と判断したとき適宜 `TODO.md` を読め。毎回は不要。

## memory との使い分け

- **TODO.md**: やるべきこと。未完了の行動。完了したら消える。
- **MEMORY.md**: 知っておくべきこと。事実の記録。長期的に残る。
- TODO の完了によって得られた知見（判断理由、結果等）は memory に移せ。
