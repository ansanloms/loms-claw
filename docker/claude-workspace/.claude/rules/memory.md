# メモリ管理

ワークスペースにファイルベースの永続メモリがある。
ディスクに保存したものだけが記憶。隠れた状態は存在しない。

## ストレージ構成

### MEMORY.md（ワークスペースルート）

長期記憶。セッションをまたいで参照する価値のある情報を保持する。

保存すべきもの:

- ユーザの好み・癖・繰り返し出る要望
- 重要な判断とその理由
- 過去の会話で確定した事実（プロジェクト方針、運用ルール等）

保存すべきでないもの:

- TODO・タスク（→ TODO.md へ）
- 一時的なタスクの進捗
- コードや git 履歴から導出できる情報
- 当日限りの文脈（→ 日次メモへ）

形式:

- 箇条書きで簡潔に。1 項目 1〜2 行。
- カテゴリで分類（例: `## 好み`, `## 判断履歴`）。
- 古くなった情報は削除または更新しろ。肥大化させるな。

### memory/YYYY-MM-DD.md（日次メモ）

当日の会話で得た情報のうち、翌日以降も参照する可能性があるものを記録する。

保存すべきもの:

- 会話中に出た決定事項
- 調査結果の要点
- 翌日に引き継ぎたい文脈

形式:

- ファイル名は `memory/YYYY-MM-DD.md`（例: `memory/2026-04-01.md`）。
- 時系列で追記。見出しで話題を区切る。

## 参照ルール

- 過去の文脈・ユーザの好み・以前の判断が必要と判断したとき、MEMORY.md や直近の日次メモを読め。
- 毎回読む必要はない。文脈から必要性を判断しろ。
- 日次メモは直近数日分を優先。古いものは必要なときだけ。

## 書き込みルール

- 今後のセッションで参照する価値があると判断したら保存しろ。
- 長期的な情報は MEMORY.md へ。当日の文脈は日次メモへ。
- MEMORY.md に書いたら、元の日次メモから重複を消せ。
- 定期的に MEMORY.md を見直し、古い・不正確な情報を整理しろ。
