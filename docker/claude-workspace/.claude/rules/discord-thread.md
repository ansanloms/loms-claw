# Discord スレッドの使い方

Discord のスレッドを能動的に立てて話題分離するための判断基準と運用方針。
技術仕様 (URL・パラメータ・curl 例) は `.claude/skills/discord/SKILL.md` を参照。

## なぜ自分でスレッドを立てるのか

bot は同一チャンネルで複数の話題が並行することがある (例: `#travel` で「明日の旅行」と「来週の旅行」)。チャンネル単位で session が共有されると話題が混ざるが、スレッドを切れば独立した session / model / effort が動く。

ユーザがスレッドを切ってくれるのを待つのではなく、自分で会話の流れを判断して `POST /discord/channels/{channel_id}/threads` か `POST /discord/channels/{channel_id}/messages/{message_id}/threads` を叩いてスレッドを立てろ。

## スレッドを立てるべきタイミング

以下のいずれかに該当したら検討する:

- 同じチャンネルで **明らかに別の話題** に切り替わった (例: 旅行の話 → コードレビューの話)
- ユーザが **特定の話題を継続的に追跡したい** と示した (例: 「この件は別で進めたい」「あとで再開したい」)
- ユーザが**長文の調査・計画を依頼** し、後でその文脈を保ったまま再開する見込みがある (登山計画、技術調査、長期タスク等)
- **依頼の規模が大きく**、副次的な質問・派生議論が発生しそう

逆に **立てなくていい** ケース:

- 一往復で終わる質問・雑談
- 既存スレッド内で会話している (= もう分離されている)
- ユーザが明示的に「ここで話して」と言った場合

迷ったらスレッドを立てない側に倒せ。後からでも作れるが、消すのは手間。

## 2 つのエンドポイントの使い分け

### `POST /discord/channels/{channel_id}/threads` (チャンネル直下)

- 自分が「ここから別話題を始めたい」と判断し、**新しい会話を切り出すケース**
- 例: ユーザの依頼に応えて長期タスクを開始する直前。「この件はスレッドに分けて進めるぞ」と宣言してから作る

### `POST /discord/channels/{channel_id}/messages/{message_id}/threads` (メッセージから派生)

- ユーザの **特定の発言を起点に話題を分離したい** ケース
- 元メッセージが thread の発端 (starter) として表示され、文脈の発端がそのまま見える
- 例: ユーザが投稿した長文の依頼や複雑な質問に対して、「このメッセージから派生させて回答するぞ」とする時

迷ったらメッセージから派生させたほうが文脈の発端が見えて後で振り返りやすい。

## スレッド名の付け方

- **話題が分かる短い名前**。15-30 文字程度を目安に
- 日付・連番を入れるな (Discord 上で時系列は自動的に分かる)
- 例: 良 →「明日の屋久島の準備」「ヤマレコ API 認証調査」/ 不可 →「スレッド 1」「議論」「2026-05-04 の話」
- 既に同じ親チャンネル内に類似名のスレッドがある場合は重複しない名前にしろ

## ユーザへの通知

スレッドを作ったら、**作ったことと作った目的**を作成元のチャンネルにも一言伝えろ。ユーザが見落として混乱するのを防ぐ。

例:

> この話題はスレッドに分けたぞ。続きは「明日の屋久島の準備」スレッドで進める。

レスポンスの `id` を `channel_id` として `POST /discord/channels/{channel_id}/messages` を叩けばスレッド側にも初手のメッセージを置ける。

## 注意

- bot ロールに **Create Public Threads** 権限が無いと 500 エラーが返る。エラーが続くようならユーザに権限付与を依頼しろ
- フォーラムチャンネル / 音声チャンネル / 既にスレッドの中では立てられない (400 が返る)。親チャンネルが普通のテキストチャンネルか確認しろ
- Private thread は現状非対応。全て Public スレッドとして作られる
- スレッド作成しただけでは現在のメッセージコンテキストは変わらない (お前は親チャンネルから呼ばれた状態のまま)。スレッドにメッセージを置きたければレスポンスの `id` を `channel_id` として `POST /discord/channels/{id}/messages` を叩け
- ユーザが次回そのスレッド内で発言した時、お前は新しいメッセージコンテキスト (= thread スコープ) で起動される。session / model / effort もスレッド側のものが使われ、`{{discord.channel.id}}` には thread ID が入る
